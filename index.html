How did ~/$ happen?

Solving course requirements I got introduced to work with Matlab. I had disks full of spread sheets which I wasn't able to maintain anymore. So I was asking myself whether I could apply Matlab on that terrain aswell.

That's not all. Is it?

No, that's where it all started. As a side quest I worked on calculating benchmarks for (financial) options. Something I intended to do. But was unable to accomplish with spread sheets and VBA.

At a later point in time I decided to offer the calculation results and reports based on the results to selected people, which they seemed to like.

It is convenient to have a set of default plots ready to consume, to share or to build upon. However, changing the question usually require a new set of plots. This in turn requires access to the underlying data. While distributing HDF5 or CSV files was not an issue, keeping track of versions would have been a nightmare. Building a RESTful web API was the solution I favored.

A prerequisite for an Web API would be database. Database, back end, front end. That's a little more than Matlab/Octave could offer. Obviously the solution had do be ... something completely different.

The ingredients?

  • cloud VMs acting as scalable (virtual) physical devices
  • Postgres to store and distribute data
  • (scientific) Python stack to shape and mold the data
  • flask as a lightweight extensible micro framework connecting front and back end
  • plotly.js / dash for reactive single page apps wrapped with Dash
